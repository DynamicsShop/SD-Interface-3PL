## SD Interface 3PL Releases

### 1.0.0

#### Enhancements

- Initial Build.

- 3rd Party Logistics Interface objects were created.

- Mapping functionality and objects were created.

- Master data functionality and objects were created.

- Transactional Outbound Data files from NAV were created for Sales Orders, Purchase Orders, Transfer Shipments, and Transfer Receipts. Transactional Data files were presented as an ODATA Query read only.

- Functionality and logic was created to place the Transactional Inbound Data files from WMS to NAV in a staging table.

- Functionality for validating, testing, processing and error handling the Inbound Transactional Data from the WMS was created. Users have three options to Process, Skip or Error Reset.

- A Role Centre for SD Stores Manager 3PL Interface was created.

- Our standard About page was created and surfaced on the SD Stores Manager 3PL Interface Role Centre and Setup List.

- Objects were reordered and renumbered.

- Activity Pages for KPIs were created and added to the Role Centre.

- Two navigation pane actions were added to the Role Centre - a Mapping action and a Master action.

- The SD-3PL RC Activities Mappings page was removed from the Role Centre as mappings were filtered by 3PL Code.

- Amended logic of SD-ST3PL in the Inbound page to cater for processing inbound entries with no serial numbers.

- Added a new field 3PL Reason Code to the SD-3PL Inbound Entries.

- Master pages were amended to be used for full or differential sequencing.

- A datetime stamp was added to the Transactional ODATA tables.

- Added a filterable datetime modified field to the Transactional tables.

- Added Lot and Serial Number to the INDEX Web Services Page.

- Standard NAV OneNote, Notes and Links were removed from the SD-3PL pages.
